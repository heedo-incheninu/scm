# Design.md

## Inventory Dashboard 리디자인 계획서

이 문서는 현재 SCM 리스크 PoC를 `Inventory Dashboard` 형태로 고도화하기 위한 디자인 계획이다. 목표는 “물류 위기 분석 앱”을 “재고 상태를 한눈에 보고 바로 조치하는 운영 대시보드”처럼 보이게 만드는 것이다.

## 참고 사이트 5개

1. [Microsoft Learn — Inventory dashboards](https://learn.microsoft.com/en-us/dynamics365/intelligent-order-management/inventory-dashboards)
   - 필터, KPI, 차트, 테이블을 한 화면에서 연결하는 재고 대시보드 구조 참고.
2. [Geckoboard — Inventory Dashboard Example](https://www.geckoboard.com/dashboard-examples/operations/inventory-dashboard/)
   - 실시간 재고 수준, 부족 위험, 주문 판단을 쉽게 보여주는 운영형 KPI 보드 참고.
3. [Retool — Inventory Management Dashboard](https://retool.com/templates/inventory-management-dashboard)
   - 재고 관리용 내부 도구 레이아웃, 데이터 테이블 중심 UI 참고.
4. [GoodData — Supply Chain Dashboard Examples](https://www.gooddata.ai/blog/supply-chain-dashboard-examples/)
   - 공급망 대시보드를 재고·물류·공급사 관점으로 나누는 정보 구조 참고.
5. [Bold BI — Supply Chain Dashboards](https://www.boldbi.com/dashboard-examples/supply-chain/)
   - inventory carrying rate, inventory-to-sales ratio, return rate 등 공급망 KPI 배치 참고.

## 리디자인 목표

- 첫 화면을 `재고 현황판`으로 바꾼다.
- 사용자는 앱에 들어오자마자 “어떤 SKU가 부족한지, 얼마가 필요한지, 지금 예산으로 어디까지 보호되는지”를 알 수 있어야 한다.
- 전문 용어보다 운영자가 바로 이해할 수 있는 문구를 우선한다.
- 3D 그래프는 상세 분석 탭에 유지하되, 홈 화면은 KPI 카드·상태 그래프·우선 조치 테이블 중심으로 단순화한다.

## 핵심 사용자 시나리오

1. 사용자가 CSV 또는 예제 데이터를 선택한다.
2. 물류 시나리오와 예산 비율을 조정한다.
3. 홈 화면에서 전체 재고 건강 상태를 확인한다.
4. 부족 위험 SKU와 예산 배정 우선순위를 확인한다.
5. 필요하면 품목 중요도, 필요 재고, 예산 추천, 전략 비교 탭에서 상세 분석을 본다.
6. AI 조언 탭에서 실행 항목을 요약한다.

## 홈 화면 정보 구조

Inventory Dashboard 홈은 다음 순서로 구성한다.

```text
상단 헤더
└─ 데이터명 / 시나리오 / 예산 / 마지막 분석 시점

KPI 카드 5개
└─ 총 SKU 수
└─ 부족 위험 SKU
└─ 안전재고 필요 금액
└─ 사용 가능 예산
└─ 중요 품목 보호 수준

메인 대시보드 영역
├─ 좌측: 재고 상태 분포 차트
├─ 중앙: 예산 배정 현황 차트
└─ 우측: 즉시 조치 카드

하단
├─ 우선 조치 SKU 테이블
└─ 시나리오 출처 / 산식 설명 / 데이터 한계
```

## 화면별 개편 방향

### 1. 한눈에 보기

현재 첫 화면을 가장 크게 바꾼다.

- `재고 현황 요약` 제목 사용
- KPI 카드 4개에서 5개로 확장
- “먼저 확인할 품목” 테이블을 `우선 조치 SKU` 테이블로 변경
- 표 위에 재고 상태 분포 차트 추가
- 예산 20% 비교 카드는 유지하되, 작은 비교 위젯으로 축소

### 2. 품목 중요도

- 기존 3D 중요도 맵 유지
- 상단에 ABC-XYZ 등급별 SKU 수 카드 추가
- 표는 등급, 매출 비중, 수요 변동계수, 우선순위 점수를 중심으로 정렬

### 3. 필요 재고

- 기존 3D 필요 재고 맵 유지
- `현재고`, `권장 안전재고`, `추가 필요 수량`을 비교하는 막대 차트 추가
- 산식 설명은 접기/펼치기 유지

### 4. 예산 추천

- 기존 3D 예산 배정 맵 유지
- 예산 사용률, 미배정 위험 금액, 완전 확보 SKU 수를 카드로 표시
- 표는 확보율이 낮고 우선순위가 높은 SKU부터 정렬

### 5. 전략 비교

- 현재 Plotly 라인 차트 유지
- X축은 10% → 100% 오름차순 고정
- 100%가 항상 가장 오른쪽에 오도록 숫자 축 사용
- 표는 예산 비율, 전략, 보호 수준, 완전 확보 SKU, 남은 금액 순서로 유지

## 홈 화면 KPI 정의

| KPI | 의미 | 표시 방식 |
| --- | --- | --- |
| 총 SKU 수 | 분석 대상 품목 수 | 숫자 카드 |
| 부족 위험 SKU | 추가 필요 수량이 0보다 큰 SKU | 숫자 + 위험 색상 |
| 안전재고 필요 금액 | 모든 SKU의 추가 확보 필요 금액 | 원화 카드 |
| 사용 가능 예산 | 사용자가 선택한 예산 비율의 금액 | 원화 카드 |
| 중요 품목 보호 수준 | 우선순위·필요 금액 가중 충족률 | 퍼센트 카드 |

## 시각화 컴포넌트 계획

- `재고 상태 분포`: 정상, 주의, 위험 SKU 개수를 도넛 또는 가로 막대 차트로 표시
- `카테고리별 필요 금액`: 카테고리별 추가 필요 예산을 막대 차트로 표시
- `예산 배정 현황`: 필요 금액 대비 배정 금액을 누적 막대 또는 waterfall 형태로 표시
- `우선 조치 SKU`: SKU명, 등급, 현재고, 권장재고, 부족 수량, 필요 금액, 확보율 표시
- `전략 비교`: 균등·비례·우선순위 배분의 보호 수준 라인 차트 유지

## 레이아웃 원칙

- 사용자는 위에서 아래로 읽으면서 `상황 파악 → 원인 확인 → 조치 결정` 순서로 이해해야 한다.
- 홈 화면에는 3D 차트를 넣지 않는다. 3D는 상세 분석용으로 사용한다.
- 첫 화면은 숫자와 상태 색상을 중심으로 빠르게 읽히게 한다.
- 모든 테이블은 SKU명과 품목 코드를 함께 표시한다.
- 위험도가 높은 SKU는 빨강, 주의는 주황, 안정은 초록으로 통일한다.

## 구현 순서

1. 홈 화면 정보 구조를 Inventory Dashboard 형태로 재배치한다.
2. KPI 계산용 보조 컬럼을 추가한다.
   - 부족 위험 SKU 수
   - 카테고리별 필요 금액
   - 재고 상태 등급
3. 홈 화면 차트를 추가한다.
   - 재고 상태 분포
   - 카테고리별 필요 금액
   - 예산 배정 요약
4. `우선 조치 SKU` 테이블 컬럼과 정렬 기준을 정리한다.
5. 상세 탭의 3D 그래프는 유지하되, 상단 설명과 카드만 Inventory Dashboard 톤으로 맞춘다.
6. 전체 CSS를 카드형 운영 대시보드 스타일로 정리한다.
7. 브라우저에서 홈, 상세 탭, CSV 예제, 업로드 흐름을 확인한다.
8. `checklist.md`에 리디자인 진행 상태를 업데이트한다.

## 완료 기준

- 홈 화면만 봐도 재고 부족 위험과 예산 보호 수준을 이해할 수 있다.
- SKU별 실제 품목명이 테이블과 차트 툴팁에 표시된다.
- 좌측 메뉴는 카드형 내비게이션을 유지한다.
- 3D 그래프는 상세 탭에서만 사용한다.
- 전략 비교의 X축은 항상 100%가 가장 오른쪽에 위치한다.
- 시나리오 출처, 산식 설명, 데이터 한계가 화면에 직접 표시된다.
