# 발표 전 피드백 반영 계획

이 문서는 AI 해커톤 발표 전 화면 완성도와 설명력을 높이기 위한 긴급 수정 계획이다.  
목표는 발표 중 오류 문구, 영어 컬럼명, 출처 불명확성, 미완성처럼 보이는 UI 요소를 제거하는 것이다.

## 우선순위 요약

| 우선순위 | 항목 | 목적 |
| --- | --- | --- |
| 긴급 | AI 조언/OpenAI 문구 정리 | 발표 화면의 API 연결 상태를 명확하게 표시 |
| 긴급 | GSCPI 오류 문구 제거 | 심사위원에게 시스템 오류가 노출되지 않도록 처리 |
| 긴급 | AI 조언 데모 출력 렌더링 수정 | 굵게 표시용 별표 같은 raw 마크다운 노출 제거 |
| 중요 | 실제 납기 기반 충격 수준 자동 진단 | 위기배수 선택 근거를 사용자가 이해 가능하게 제공 |
| 중요 | 예산 추천 테이블 한글화 | UI 언어 일관성 확보 |
| 텍스트 | 1.8x·2.2x 출처 문구 수정 | 검증되지 않은 단일 출처 표현 제거 |

## 긴급-1. AI 조언 메뉴와 사이드바 OpenAI 문구 정리

### 적용 위치

- AI 조언 페이지 전체
- 사이드바 하단
- 도움말 페이지 하단

### 현재 상태

- 사이드바에 `OpenAI 계정 연결` 버튼 또는 영역이 표시된다.
- AI 조언 페이지 하단에는 `OpenAI API 연결 완료 · 모델: gpt-5.5` 문구가 표시된다.
- 도움말 하단에 `OpenAI 연결 안내` 섹션이 표시된다.

### 수정 내용

- `OpenAI API 연결 완료 · 모델: gpt-5.5` 문구는 그대로 유지한다.
- 사이드바의 `OpenAI 계정 연결` 버튼 또는 expander 영역은 삭제한다.
- 도움말의 `OpenAI 연결 안내` 섹션은 삭제한다.

### 완료 기준

- 사이드바에 OpenAI 계정 연결 버튼이 보이지 않는다.
- AI 조언 페이지에는 연결 완료 문구만 남는다.
- 도움말 페이지에는 OpenAI 연결 안내 섹션이 보이지 않는다.

## 긴급-2. 위험 시나리오 메뉴 GSCPI 오류 문구 제거

### 적용 위치

- 위험 시나리오 페이지 상단

### 현재 상태

화면에 다음과 같은 오류 문구가 그대로 노출될 수 있다.

```text
GSCPI 지수 조회 실패: Missing optional dependency 'xlrd'...
```

### 수정 내용

GSCPI 기능은 발표 화면에서 제거한다. 오류 문구 대신 아래 안내 문구를 표시한다.

```text
시나리오를 직접 선택하거나 아래 실제 납기를 입력하면 자동으로 충격 수준을 계산합니다.
```

### 완료 기준

- 위험 시나리오 페이지에서 GSCPI, xlrd, 자동 조회 실패 문구가 보이지 않는다.
- 사용자는 수동 시나리오 선택 또는 실제 납기 입력 방식만 보게 된다.

## 긴급-3. AI 조언 데모 모드 마크다운 렌더링 수정

### 적용 위치

- AI 조언 페이지
- 데모 모드 ON
- `AI 조언 만들기` 버튼 클릭 후 출력 영역

### 현재 상태

출력 텍스트에 굵게 표시용 별표 같은 마크다운 문법이 raw 문자로 노출된다.

### 수정 내용

기본 수정 방향은 출력 방식을 `st.markdown()`으로 변경하는 것이다.  
단, 화면 구조상 `st.write()` 또는 HTML 컨테이너를 유지해야 한다면 AI 응답 생성 프롬프트에 아래 지시를 추가한다.

```text
마크다운 문법을 사용하지 말고 plain text로만 출력하라.
```

### 완료 기준

- 발표 화면에 의도하지 않은 raw 마크다운 문자가 노출되지 않는다.
- SKU명과 조언 문장이 자연스럽게 렌더링된다.

## 중요-1. 실제 납기 입력 기반 충격 배수 자동 진단

### 적용 위치

- 위험 시나리오 페이지
- 기존 GSCPI 영역 대체 위치

### 추가 입력 필드

```text
현재 공급사 또는 포워더에게 확인한 실제 예상 납기 (일 단위 입력)
```

### 자동 계산 기준

- 기준 리드타임: `44일`
- 기준 설명: `World Bank 2023 평균 리드타임 44일`
- 계산식:

```text
충격 배수 = 입력한 실제 예상 납기 ÷ 44
```

### 시나리오 매핑

| 계산 배수 | 안내 문구 | 자동 선택 시나리오 |
| --- | --- | --- |
| 1.0 이상 1.1 이하 | 평상시 수준으로 계산합니다 | `normal`, `1.0x` |
| 1.1 초과 1.3 이하 | 중간 충격 수준으로 계산합니다 | `moderate`, `1.18x` |
| 1.3 초과 2.0 이하 | 고충격 수준으로 계산합니다 | `high`, `1.8x` |
| 2.0 초과 | 극단 충격 수준으로 계산합니다 | `extreme`, `2.2x` |

### 완료 기준

- 사용자가 실제 예상 납기를 입력하면 충격 배수가 자동 계산된다.
- 계산 결과에 따라 시나리오가 자동 선택되거나, 최소한 추천 시나리오가 명확히 표시된다.
- 발표 중 `1.8x`와 `2.2x` 선택 근거를 “공급사 납기 입력 기반 자동 진단”으로 설명할 수 있다.

## 중요-2. 예산 추천 메뉴 knapsack 결과 테이블 컬럼명 한국어 변환

### 적용 위치

- 예산 추천 페이지 하단
- `기존 우선순위 수량 배분도 함께 보기` 펼침 테이블

### 현재 상태

일부 컬럼명이 영어로 노출된다.

### 컬럼명 변환 기준

| 현재 컬럼명 | 표시 컬럼명 |
| --- | --- |
| `sku_id` | 품목코드 |
| `abc_xyz` | 관리등급 |
| `required_budget` | 필요예산 |
| `allocated_budget` | 배정예산 |
| `allocated_quantity` | 배정수량 |
| `unmet_quantity` | 미충족수량 |
| `fulfillment_rate` | 충족률 |

### 완료 기준

- 펼침 테이블에 위 컬럼명이 모두 한국어로 표시된다.
- 예산 추천 화면의 다른 한국어 UI와 언어 톤이 일관된다.

## 텍스트-1. 도움말 메뉴 1.8x·2.2x 출처 표기 수정

### 적용 위치

- 도움말 페이지
- 물류 시나리오 근거 섹션

### 현재 문구

```text
고충격 1.8배 — 과거 조사 기록 기반 표현
극단충격 2.2배 — 과거 조사 기록 기반 표현
```

### 수정 문구

```text
고충격 1.8배 — 업계 맥락 기반 시나리오 입력값. 사용자가 상황에 따라 조정 가능
극단충격 2.2배 — 업계 맥락 기반 시나리오 입력값. 사용자가 상황에 따라 조정 가능
```

### 하단 소주 추가

```text
고충격·극단충격 배수는 검증된 단일 출처가 없어 시나리오 입력값으로 처리합니다.
```

### 완료 기준

- 도움말 페이지에서 과거 조사 기록을 직접 출처처럼 표시하지 않는다.
- 1.8x와 2.2x는 검증된 고정값이 아니라 조정 가능한 시나리오 입력값으로 설명된다.

## 검증 계획

개발 완료 후 아래 순서로 확인한다.

1. `uv run streamlit run app.py`로 로컬 앱을 실행한다.
2. 사이드바에서 `OpenAI 계정 연결` 버튼 또는 영역이 사라졌는지 확인한다.
3. AI 조언 페이지에서 `OpenAI API 연결 완료 · 모델: gpt-5.5` 문구가 유지되는지 확인한다.
4. AI 조언 데모 모드 출력에 raw 마크다운 문자가 보이지 않는지 확인한다.
5. 위험 시나리오 페이지에서 GSCPI 오류 문구가 보이지 않는지 확인한다.
6. 실제 납기 입력값 `44`, `52`, `80`, `100`일을 넣어 시나리오 진단 결과가 의도대로 표시되는지 확인한다.
7. 예산 추천 페이지의 펼침 테이블 컬럼명이 한국어로 표시되는지 확인한다.
8. 도움말 페이지에서 1.8x·2.2x 설명이 시나리오 입력값 기준으로 수정되었는지 확인한다.

## 개발 시 주의사항

- 이번 피드백은 발표 안정성이 목적이므로, GSCPI 내부 모듈 삭제보다 화면 노출 제거를 우선한다.
- 안전재고 계산식과 위기배수 적용 방식은 이미 수정된 기준을 유지한다.
- OpenAI API 크레딧이 없거나 호출이 실패해도 데모 모드는 정상 동작해야 한다.
- UI 문구는 일반 비전문가가 이해할 수 있는 한국어를 우선 사용한다.
